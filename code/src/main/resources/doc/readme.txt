1. 开发框架结构
/src
    /main/              正式的开发目录
        /java              代码目录
            .../base            公用的、基础的放这里
            .../common          公用的、基础的放这里
            .../manage          系统框架相关的，非业务的，开发时无须关注，有时间可以加强优化
            .../user            企业、用户
            .../scenic          景区
            .../captial         资金
            .../product         产品
            .../orders          订单
            .../report          报表
            ...XXX              注：/sale下的每个目录都可看作是一个“业务模块”，有新的业务模块就新增，方便以后做服务拆分
        /resources      资源文件目录  
            /conf               基础（公用）配置
            /db                 数据库配置信息（dev：开发、test：测试、pro：生产）
            /doc                文档
            /mybatis            mybatis的配置
            /spring             spring的配置：applicationContext.xml 是主文件，引用其它的
            /sql                数据库脚本
            /template           模板
            logback.properties  日志相关
            logback.xml         日志相关
        /webapp
            /WEB-INF
                /jsp            页面
                /lib            jar包
                /static         静态文件
                    /images
                    /css
                    /js
                /tld            JSP自定义标签
                decorators.xml      sitemesh页面装饰器配置文件
                web.xml             
/target                         编译、打包后的的存放目录
/pom.xml                        Maven项目的配置文件

程序运行及示例Demo
    1. 把项目导入到工作空间下。
    2. 把/resoruce/doc/utils目录放到本机Maven仓库的com/coolshow/目录下（没有就新建）
    3. 修改本机MySql的用户/密码信息为：root/123456，端口为：3306  
                注：如不想修改本机MySql配置，可修改：/resource/db/dev.properties文件中内容。建议开发人员环境尽量相同！
    4. 打开MySql，导入/resource/sql/1_yt_sale_manager.sql脚本（其它的不需要管，也不要改！）
    5. 打开浏览器，访问服务。登录账号：admin / Cs123456
